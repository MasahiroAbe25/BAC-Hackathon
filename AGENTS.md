# AGENTS.md

## このプロジェクトについて

就活アプリ「BaseMe」の新機能プロトタイプ「アイデンティティポスター(メモリーツリー)」を構築するプロジェクトです。
以下の仕様に基づき、独立したWebアプリ(BaseMeの既存コードベースは使用しない)としてプロトタイプを実装してください。
6/14のチーム発表で使用するデモであり、完成度よりも「コンセプトが動いて伝わること」を優先します。

実装を始める前に、本ドキュメント全体を読み、不明点があれば実装前にまとめて質問してください。
仕様に明記がない細部(配色・フォント・余白など)はCodexの判断で構いません。

---

## 1. 機能概要

ユーザーが受けた「就活占い」の診断結果(性格特性)を中心ノードとしたツリー(マインドマップ)を表示し、

- ユーザーが「覚えたこと・興味・弱点」を自由入力すると、テキストマイニングによってトピックを抽出し、関連するブランチに新しいノード(枝)を追加する
- AIキャラクター「ケン」とのチャット対話を通じて、言語化が苦手なユーザーでも同様にトピックを抽出・追加できる
- どのブランチにも明確に関連しないトピックは、ツリー外周に浮遊するノードとして表示する

という体験を提供する。ノードが追加されるたびにツリーが「ニョキニョキ」と成長していく見え方を重視する。

---

## 2. 提供データ

`src/data/diagnoses.json` に18種類の診断結果データを配置済み(または配置すること)。各診断は以下の構造を持つ。

```json
{
  "id": "data-driven",
  "title": "データで勝つ体質",
  "summary": "診断結果の説明文",
  "branches": [
    {
      "id": "weapon",
      "category": "強み",
      "label": "数字と根拠で語る分析力",
      "keywords": ["データ", "分析", "論理", ...]
    },
    {
      "id": "growth",
      "category": "成長ポイント",
      "label": "...",
      "keywords": [...]
    },
    {
      "id": "workstyle",
      "category": "働き方",
      "label": "...",
      "keywords": [...]
    }
  ]
}
```

初期画面で18診断から1つをユーザーが選択することをアプリの起点とする。これが「自分の就活占いの結果」に相当し、選択後にその診断を中心ノードとしたツリーが表示される、というデモフローとする。

## 3. 技術スタック

- フレームワーク: React(Vite推奨)
- ツリー描画: react-flow
- レイアウト計算: d3-force, d3-hierarchy
- 形態素解析: kuromoji.js
- LLM API: Anthropic API(ケンとの対話に使用)
- 言語: TypeScript推奨(JavaScriptでも可)

---

## 4. 処理パイプライン

### 4-1. 自由入力 → テキストマイニング経路

1. ユーザーが「覚えたこと・興味・弱点」をテキストボックスに入力し、「分析」ボタンを押す
2. kuromoji.jsで形態素解析し、名詞・固有名詞を抽出する
3. 頻度ベース(またはTF-IDF)で主要キーワードを絞り込む
4. 抽出されたキーワード群を `{ label, tags }` 形式に正規化する(`label` は入力テキストの簡潔な要約、`tags` は抽出キーワード)
5. 4-2と共通の「ブランチ判定」処理に渡す

### 4-2. ケンとの対話経路

1. ユーザーがケンとのチャットUIで対話する
2. システムプロンプトには、ユーザーの中心ノード情報(診断タイトル・summary・各ブランチのlabel)を変数として埋め込み、その文脈に沿った質問を生成させる
3. 対話終了時、LLMに会話内容を要約させ、`{ topics: [{ label, tags }, ...] }` 形式のJSONで出力させる
4. 各topicについて、4-1と共通の「ブランチ判定」処理に渡す

### 4-3. ブランチ判定(共通処理)

- 比較対象は、**ユーザーが起点画面で選択した診断(`diagnoses.json`内の1オブジェクト)が持つ`branches`配列の3要素(強み/成長ポイント/働き方)のみ**。他の17診断の`branches`とは比較しない
- 各 `topic.tags` と、3ブランチそれぞれの `keywords` との **Overlap coefficient** を計算する

  ```
  overlap(A, B) = |A ∩ B| / min(|A|, |B|)
  ```

- Overlap coefficientは入力タグ数による正規化が計算式に組み込まれているため、入力の長さ(コンテキスト量)に応じて閾値を変動させる必要はない。閾値は固定値 `0.5` とする
- 3ブランチの中で最高スコアが0.5以上 → そのブランチに `leaf` ノードとして接続
- 3ブランチすべてが0.5未満 → `floating` ノードとしてツリー外周に配置

## 5. データ構造(ツリーノード)

```ts
type NodeType = "center" | "branch" | "leaf" | "floating";
type Source = "mining" | "ken_dialogue";

interface TreeNode {
  id: string;
  type: NodeType;
  label: string;
  tags?: string[];
  source?: Source;
  parentBranchId?: string | null; // floatingはnull
  similarity?: number; // ブランチ判定に使ったスコア(デバッグ表示用)
  createdAt: string;
}
```

## 6. ツリーのレイアウト・描画(react-flow)

### 6-1. 初期配置(固定)

- 中心ノード: `(0, 0)`
- 3ブランチ(強み/成長ポイント/働き方): 中心から半径150pxの位置に、120度間隔で固定角度配置(例: -90°, 30°, 150°)。`fx`/`fy` で位置を固定し、シミュレーション対象から外す

### 6-2. リーフ・浮遊ノードの配置(d3-force)

以下のフォースを組み合わせて配置を計算する。

- `forceLink`: リーフノードを親ブランチに引き寄せる(distance: 60, strength: 0.3程度)
- `forceCollide`: ノード同士の重なりを回避(半径40px程度)
- `forceRadial`: 中心からの目標距離を設定
  - `leaf`: 半径220px程度
  - `floating`: 半径320px程度(ツリー外周)

### 6-3. 新規ノード追加時の挙動

1. 新しいノードを `fx`/`fy` なしで追加する
2. シミュレーションを数十ティック分進める
3. 収束後、`fx`/`fy` を設定して位置を固定し、シミュレーションを停止する
4. react-flowの `setNodes` で座標を反映し、CSS transitionで滑らかに移動させる
5. 新規ノードはscale + fadeインのアニメーションで表示する(「ニョキニョキ」感)

### 6-4. 浮遊ノードの見た目

- 半透明・淡い色の円形ノードとして表示する
- CSS `@keyframes` で、確定した基準位置を中心に微小な往復アニメーション(漂う表現)を付与する

## 7. UI構成(画面イメージ)

- **起点画面(診断結果選択)**: 18診断の一覧から、ユーザーが自分の「就活占い」の結果を1つ選択する画面。選択肢には診断の`title`を表示する。選択後、「あなたの診断結果」という見出しと、選択した診断の`summary`を表示してから、ツリー画面に遷移する
- メイン画面: react-flowで描画されたツリーを中央に表示
- 入力エリア: 「覚えたこと/興味/弱点」を入力するテキストボックス + 「分析」ボタン
- ケンとの対話エリア: チャットUI(吹き出し形式)。診断結果の情報を踏まえた質問をケンが発話し、ユーザーが返答する

画面レイアウトの詳細(配置・配色・タイポグラフィ)はCodexの判断で構わないが、BaseMeのテーマカラー(`#544CD6` 系の紫)を基調とすることを推奨する。

## 8. 優先度・スコープ

以下の順で実装を進めること。4つすべてが優先実装対象であり、時間が不足する場合は内部実装(キーワード抽出の精度やレイアウトの細かい調整)を簡略化すること。

1. **最優先**: 診断データ読み込み → 起点画面(診断結果選択・summary表示)→ 中心ノード+3ブランチの初期表示(react-flow + 固定角度配置)
2. **優先**: 自由入力 → kuromoji.js形態素解析 → ブランチ判定 → ノード追加(d3-forceレイアウト + アニメーション)
3. **優先**: 浮遊ノードの表示・漂うアニメーション
4. **優先**: ケンとの対話UI + LLM APIによるトピック抽出 → 4-3の共通処理への合流

### 8-1. 時間があれば対応する改善・検討事項

上記4項目が完了した上で時間が余った場合に、以下の改善に着手すること。優先順位は記載順。

- **浮遊ノードがツリー本体に吸い込まれる演出**: floatingノードが、後から関連するブランチが見つかった場合に、外周から本体へアニメーションで移動する表現
- **キーワード辞書の弁別力チューニング**: 「裁量」「企画」「NPO」など複数診断で共通するキーワードが多く、ブランチ判定の精度に影響する可能性がある。`diagnoses.json`の`keywords`に、各診断固有のキーワードを追加するなどの調整
- **ケンのシステムプロンプトのチューニング**: 対話で抽出されるトピックの粒度・タグの質を見ながら、システムプロンプトの文言を調整する

## 9. デザイン・配色について

参考にしている元アプリ(BaseMe / `https://baseme.app/chats/new`)から確認できる要素は以下の通り。これらの「雰囲気」を踏襲した、オリジナルのデザイントークンを使用する(ロゴ・キャラクター画像・CSSの直接流用は禁止)。

### 9-1. 確認できる元アプリの特徴

- テーマカラー: `#544CD6`(紫〜インディゴ系)。サイトの`theme-color`として設定されている
- AIキャラクター「Ken」: サムズアップする丸みのあるイラストキャラクターがトップに配置されている。フレンドリーで親しみやすいトーン
- ナビゲーション構成: 「チャット / プロフィール / スカウト / さがす」の4項目+ログイン/新規登録ボタン
- コピーのトーン: 「就活のことなら何でも聞いて」のような口語的・カジュアルな見出し、`#就活特化型AI` `#最新版モデル` のようなハッシュタグ形式の機能訴求
- 全体として「堅すぎないAIパートナー」という印象を前面に出している

### 9-2. プロトタイプ用デザイントークン(オリジナル)

上記の印象を踏まえ、本プロトタイプでは以下のオリジナルパレット・トークンを使用する。

**カラーパレット**

| 用途 | カラーコード | 説明 |
| --- | --- | --- |
| プライマリ | `#5B4FE0` | メインの紫(元アプリのテーマカラーに近いが、独自に調整した値) |
| プライマリ(濃) | `#3F35A8` | ホバー・強調表示 |
| プライマリ(淡) | `#EFEDFF` | 背景・カード地色 |
| アクセント(暖色) | `#FFB35C` | 「ニョキニョキ」成長を示す新規ノードのハイライト等 |
| 背景 | `#FAFAFC` | ページ全体の背景 |
| テキスト(主) | `#2B2B36` | 本文・見出し |
| テキスト(副) | `#8A8A99` | キャプション・補足 |
| 浮遊ノード | `#C9C4F7`(40%透明度) | floatingノードの淡い紫 |

ツリーのノード種別ごとの配色案:

- 中心ノード: プライマリ(`#5B4FE0`)の塗り、白文字
- ブランチノード(強み/成長ポイント/働き方): プライマリ(淡)の塗り、プライマリ文字。3ブランチをカテゴリ別に色相をわずかに変える(例: 強み=紫系、成長ポイント=青紫系、働き方=ピンク紫系)のも可
- リーフノード(mining/ken_dialogue): 背景色の塗り、プライマリのボーダー
- 浮遊ノード: 半透明の淡い紫の円、ボーダーなし

**タイポグラフィ**

- 見出し・本文ともに丸みのあるゴシック体を基調とする(例: "Zen Maru Gothic", "M PLUS Rounded 1c" などのGoogle Fontsを利用可。フォントの太さは2段階程度に絞る)
- 見出しは太め、本文は通常太さ。行間はやや広め(1.6〜1.8)に取り、親しみやすさを出す

**コピー・トーンのガイドライン**

- ボタンやラベルは口語的・カジュアルな表現を使う(例:「分析する」→「もっと深掘りしてみる」、「送信」→「ケンに聞いてみる」など)
- ケンとの対話UIでは、ケンの発話に絵文字や柔らかい語尾(「〜だね」「〜してみよう」)を使い、フレンドリーな印象を出す
- ハッシュタグ的な機能訴求(`#〜`形式)は、プロトタイプのヒーロー部分やツリーの説明文に軽く取り入れても良い

### 9-3. レイアウトの型(参考)

- ヘッダー: ロゴ(オリジナル)+ナビゲーション(プロトタイプでは「ツリー」「ケンと話す」の2項目で十分)
- メインビジュアル: キャラクター(Kenに相当するオリジナルキャラクター、またはシンプルなアイコン)+短いキャッチコピー
- メインコンテンツ: react-flowによるツリー表示を中央に配置し、上記のヒーロー的な要素は控えめにする(ツリー自体が主役のため)
